# Product

## Register

product

> 注：本项目主体是应用（product register）。但架构图海报、开源主页这类"设计即产品"的独立页面，按任务临时切换到 brand register 处理。

## Users

- **MVP 阶段**：技术型二次元用户——会配 API key、装 MCP server 的开发者/极客，同时是动漫爱好者。他们在电脑前工作，随手用手机接续对话；对工具的审美有要求，反感"AI 生成感"的界面。
- **泛化阶段**：泛二次元用户（不懂技术的动漫爱好者），需要一键化的工具市场（后期目标）。

## Product Purpose

开源的本地优先 AI Agent 应用：用户上传任意图片自动转成黑白线稿风的二次元角色形象，Agent 联网搜集该角色的性格与口癖进行扮演；通过 MCP 与 Skill 双通道扩展能力；电脑做主机、手机远程接入。生产力为骨、人设为皮。成功的样子：技术用户三分钟内让"她"开口说话，并愿意把调教好的角色卡回流到社区库。

## Brand Personality

**手绘的、冷静的、有劲道的。**

青年漫画的底子（细线条、克制网点、大留白、文献感），少年漫画的点睛（标题有劲道、关键处敢盖章）。像一本装帧考究的漫画单行本：翻开是安静的分镜，扉页是压得住的标题。界面元素——分镜格、对话气泡、手绘边框、朱红印章——是功能性的漫画语言，不是贴上去的装饰。

## Anti-references

- **SaaS 模板感**：紫蓝渐变、卡片堆叠、Inter 字体、hero-metric 布局——千篇一律的 AI 官网长相。
- **赛博霓虹二次元**：紫粉色、发光描边、机能风 UI——市面上 AI 伴侣产品的主流长相，坚决不像。
- **低年龄向卡通风**：圆润糖果色、大眼睛 Q 版——幼齿，不符合技术用户定位。
- 编辑部杂志风（display serif + 斜体 + 分栏线）——那是另一条已经饱和的赛道，不是漫画。

## Design Principles

1. **线稿即品牌**：黑墨、白纸、网点灰是全部底色；彩色只有一枚朱红印章的份额，盖在最重要的地方。
2. **漫画语言承载信息**：分镜格是分区、气泡是描述、印章是强调、手绘箭头是数据流——每个漫画元素都有信息职能，去掉它信息就缺失。
3. **纸面感**：真白纸（非米色/奶油色）、实墨线（非阴影）、直角与手绘抖动线（非大圆角）。
4. **先骨后皮**：层级、留白、对齐先立住，趣味元素后上；克制处安静，点睛处敢重。
5. **开源的体面**：WCAG AA、减动态适配、快加载——体面是开源项目的门面。

## Accessibility & Inclusion

- WCAG AA：正文对比度 ≥4.5:1，大字 ≥3:1；朱红仅用于大号/加粗文字与图形元素。
- 所有动效提供 `prefers-reduced-motion` 降级。
- 键盘可达：交互元素可聚焦，hover 联动同时响应 focus。
