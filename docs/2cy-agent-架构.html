一个角色，两副面孔：既陪你聊，也替你干

三个场景共享同一个角色——先共创出"她"，然后闲聊与干活都不出戏。

角色共创 · 上传图 → 转线稿 → 角色卡微调 陪伴闲聊 · 以角色人格日常互动 任务执行 · 调工具跑技能完成真实任务

人格注入 × 本地图像 × 可插拔工具运行时

04
Data · 数据层

数据在用户设备，壁垒在社区仓库

本地优先存储保隐私，社区角色卡库做跨用户沉淀——开源项目唯一带得走的数据资产。

用户上传图片 · 形象源 公开角色百科 · 萌娘百科 / Bangumi / Fandom 角色卡资产 · 结构化人设 · 开放格式 社区角色卡共享库 · GitHub 承载 · 一键导入 本地会话历史 MCP / Skill 配置与包 开放模型权重 · HuggingFace 首启拉取
↓ Sediment · 沉淀流
↓Data → Capability → Scenario

公开角色百科经人设搜集与人格注入，变成角色共创场景里那张"会说话"的角色卡——干活闲聊都不出戏。

↓Data → Capability

开放模型权重首启自动拉取，驱动本地线稿转换——任意彩图落地成统一黑白调性，CPU 即可运行。

↓Data → Capability → Scenario

社区角色卡库通过一键导入进入共创场景——新用户零配置即获得社区调教成熟的角色。

↑ Backflow · 回流
↑Scenario → Data

用户微调后的角色卡从共创场景回流社区共享库——越多人用，库越丰富，生态越厚。

↑Scenario → Data → Capability

对话中的纠偏（口癖跑偏、性格失真被用户修正）迭代角色卡，反哺人格注入——扮演越来越准。

↑Touchpoint → Data

GitHub 社区以 PR 形式回流 Skill、MCP 配置与角色卡——开源入口同时是数据资产的进水口。

02 / Architecture Insights 架构洞察 · 壁垒、替代风险与深挖方向
Moat · 最有壁垒

数据层 · 社区角色卡库

本地优先架构下，它是唯一能跨用户沉淀的资产。角色卡随使用被不断调准并回流社区——fork 拿得走代码，拿不走生态。开放格式 + GitHub 承载，成本极低而复利极高。

Risk · 最易替代

能力层 · BYOK 模型底座

对话、工具调用来自第三方大模型，线稿转换用的是开放权重——任何同类产品都能获得同款底座。能力层自身无壁垒，必须靠场景体验与数据生态在上下两层构筑防御。

Bet · 可深挖方向

场景层 · 角色共创首体验

"上传任意图 → 变成会扮演的角色"是天然的传播引爆点。把首次体验打磨到极致——转线稿质量、人设搜集准确度、三分钟内见到"她"开口——决定开源冷启动的成败。
