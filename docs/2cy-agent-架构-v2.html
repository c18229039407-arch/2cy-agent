第 01 格

触达层

TOUCHPOINT

电脑是主机，手机是延伸，GitHub 是入口。

本地优先的混合形态：Agent 与数据都在用户电脑上，手机与社区只是通向它的两条路。

桌面应用 本地服务 + 内嵌 Web UI 手机 H5 远程接入 配对码 / 可自托管中转 GitHub 开源仓库 分发与社区入口
第 02 格

场景层

SCENARIO

一个角色，两副面孔——既陪你聊，也替你干。

三个场景共享同一个角色：先共创出「她」，然后闲聊与干活都不出戏。

角色共创 上传图 → 转线稿 → 角色卡微调 陪伴闲聊 以角色人格日常互动 任务执行 调工具、跑技能，完成真实任务
第 03 格

能力层

CAPABILITY

人格注入 × 本地图像 × 可插拔工具运行时。

扩展接入下沉于此：MCP 与 Skill 不是场景，而是能力的可插拔来源。

BYOK 模型抽象层 Claude / OpenAI 协议 人格注入引擎 角色卡 → System Prompt 联网人设搜集 Web 搜索 + 结构化抽取 线稿转换 本地轻量模型 CPU 可跑 / SD 重绘可选 MCP 宿主 stdio / HTTP Skill 运行时 链接 / skill.md 拉取加载 跨端同步与中转 对话同步 + 本地工具降级
第 04 格

数据层

DATA

数据在用户设备，壁垒在社区仓库。

本地优先存储保隐私，社区角色卡库做跨用户沉淀，这是开源项目唯一带得走的数据资产。

用户上传图片 形象源 公开角色百科 萌娘百科 / Bangumi / Fandom 角色卡资产 结构化人设 · 开放格式 社区角色卡共享库 GitHub 承载 · 一键导入壁垒 本地会话历史 MCP / Skill 配置与包 开放模型权重 HuggingFace 首启拉取
沉淀下层支撑上层
↓数据 → 能力 → 场景

公开角色百科经人设搜集与人格注入，变成共创场景里那张「会说话」的角色卡，干活闲聊都不出戏。

↓数据 → 能力

开放模型权重首启自动拉取，驱动本地线稿转换：任意彩图落成统一的黑白调性，CPU 即可运行。

↓数据 → 能力 → 场景

社区角色卡库一键导入进入共创场景，新用户零配置，直接获得社区调教成熟的角色。

回流使用反哺下层
↑场景 → 数据

用户微调后的角色卡从共创场景回流社区共享库：越多人用，库越丰富，生态越厚。

↑场景 → 数据 → 能力

对话中的纠偏（口癖跑偏、性格失真被用户修正）迭代角色卡，反哺人格注入，扮演越来越准。

↑触达 → 数据

GitHub 社区以 PR 形式回流 Skill、MCP 配置与角色卡：开源入口同时是数据资产的进水口。
